# S3

## S3 서버 액세스 로그 vs CloudTrail

S3 서버 액세스 로그 : S3 버킷에 저장된 데이터의 객체 수준에 대한 가시성 있는 로그 제공. 기능 자체는 무료이나, 로그를 남기기 위한 버킷이 따로 생성되어야 하고, 해당 버킷에 저장되는 로그의 스토리지 요금이 발생함.

CloudTrail : 같은 용도로 사용할 수 있지만, S3로 전송되는 모든 요청에 대한 정보를 제공하지는 않음. 지원하지 않는 레코드가 있으므로 S3 전반에 대한 모든 로깅이 필요하다면 S3 서버 액세스 로그를 사용.

# EBS

- HDD 유형은 부팅 가능한 볼륨으로 사용 불가능함. SSD는 부팅볼륨으로 사용할 수 있으며, 처리 속도가 빠르기 때문에 단순 저장보다는 많은 트랜잭션이 발생할 수록 적합함.
- "EBS 프로비저닝 된 IOPS SSD(io1,io2)" 는 io 비용이 추가되어 범용 SSD보다 2~3배 가량 비쌈. 지속적인 IOPS 성능보장을 요구하는 애플리케이션에 적합한 케이스이며 비용효율적이라고 할 수 는 없음.
